## What a financial advisor one-pager actually looks like

A CFP's one-page client plan is:
- **Typography-first.** The hierarchy is established through type size and weight, not color blocks or icons. Section headers are large and clear; data sits beneath them with obvious labels.
- **Vertical flow.** The reader moves down the page through sections: goals → current situation → allocation → action plan. No lateral scanning, no sidebar.
- **Sparse data visualization.** One allocation chart. One summary table. No more — every visual element earns its space.
- **White space is load-bearing.** Breathing room between sections signals that each section is a complete thought, not part of a list.
- **Warm paper-like background.** Off-white or cream — not clinical white. Conveys that this was prepared for you, not generated for anyone.
- **Color used for emphasis only.** One accent color for highlights, one color for positive/negative status. Not a color per data category.
- **Copy is authoritative but plain.** "Your current allocation is 60% equity" not "Explore your portfolio diversity." No jargon. No hedging.

## How this maps to the app

| One-pager element | App equivalent |
|---|---|
| Vertical scroll through sections | Mobile viewport = the vertical canvas. Dashboard scrolls through: Health tier → allocation donut → nudge → (below fold) member summary |
| Section headers as primary hierarchy | Each dashboard section has a clear label — "Your Household Health", "Where your money lives", "Next step" — not icon-only |
| Single allocation chart | The donut is the only chart on the home screen |
| Warm off-white paper background | Base background: warm off-white (not pure white, not dark) with dark text |
| Color for emphasis, not decoration | Deep teal accent for primary actions; semantic amber/teal/green for tier status only |
| Authoritative plain copy | Nudge copy sounds like advice from a person who knows your situation, not a system error |

## The negative constraint

**What makes a fintech app look AI-generated / prompted / generic:**
- Gradient cards (glassmorphism, frosted panels)
- "Green = money" applied everywhere with no semantic meaning
- A dashboard that tries to show 12 things at once — widget soup
- Thin sans-serif font at huge size on a very light background (the "premium fintech" template)
- Coin, graph, or upward-arrow stock illustrations
- Rounded rectangle cards with drop shadows stacked uniformly on a gray background
- Copy that sounds like a chatbot: "Let's get started on your financial journey!"

**The active prevention:** Design each screen as a section of a vertical document, not a widget dashboard. Ask: "Would a CFP print this section in their client plan?" If yes, the density and layout are right. If not, it's probably too widget-y.
